# Coffee — Детекция (CV)

Обнаружение бумажного стакана с кофе в кадре, выдача 3D-координат для манипулятора.

## Стек
- YOLOv8 / RT-DETR (ultralytics) — детекция объектов
- OpenCV — работа с изображениями
- pyrealsense2 — камера RealSense D435 (RGB + глубина)

## Установка
```bash
cd coffee/detection
pip install -r requirements.txt
```

## Этапы

### 1. Тест готовой модели YOLO (baseline)
```bash
python baseline_test.py --images test_images/ --class-id 41  # COCO "cup"
```
Оценить mAP и визуально — отличает ли бумажный стакан от кружки.

### 2. Сбор датасета (если готовая модель плохо работает)
- 200–500 фото бумажных стаканов с кофе (разные бренды: Starbucks, кофейни, plain white)
- Разметка: `labelme` или `roboflow`, формат YOLO
- Аугментации: освещение, ракурс, наличие пара, разные фоны

### 3. Дообучение (fine-tune)
```bash
python finetune.py \
  --data dataset.yaml \
  --model yolov8s.pt \
  --epochs 100 \
  --imgsz 640
```

### 4. Инференс + 3D-локализация
```bash
python infer.py --camera realsense --model best.pt
```
На выходе: bounding box + 3D-координаты центра стакана в системе координат камеры → пересчёт в базовую систему робота через TF.

## Лёгкий режим (без YOLO)

Если `ultralytics` установить нельзя (нет места под torch / нет GPU /
окружение без сети), используйте `baseline_light.py` — детектор на чистом
OpenCV без нейросетей. Требует только `opencv-python` (или
`opencv-python-headless`), `numpy` и `rich`.

### Алгоритм
1. **HSV-сегментация** по 3 цветовым диапазонам:
   - бежевый/кремовый (бумажный стакан)
   - коричневый (тёмные/«кофейные» стаканы)
   - белый бумажный (низкая насыщенность, высокая яркость)
2. **Морфология**: closing (заполняет дыры внутри стакана) + opening
   (убирает мелкий шум).
3. **findContours** → фильтр по площади (`>500 px²`), доле кадра (`<45%`),
   границам (отсев компонентов, касающихся 3+ краёв кадра — это фон).
4. **Слияние вертикальных фрагментов**: если cup разрезан тёмным sleeve'ом
   на 2 части — они объединяются в один bbox.
5. **Проверка формы**: aspect ratio `w/h` в `[0.4, 2.0]`, эвристика
   «стакан вытянут вверх» (`h/w ≥ 0.6`).
6. **Edge-based fallback**: если цветовая маска покрывает `>40%` кадра —
   фон совпал со стаканом по цвету, переключаемся на Canny + dilate
   + закрытые контуры с проверкой «стаканности» внутренней области.
7. **NMS** (IoU `0.25`) для подавления дубликатов.
8. **Confidence** = `sqrt(area/img_area) × density_mask_in_bbox`.

### Запуск
```bash
# Сгенерировать синтетический тестовый набор:
python ../../scripts/gen_synthetic_cups.py --count 50 --seed 42

# Запустить детектор:
python baseline_light.py --images test_images/ --visualize
# Аннотированные изображения сохранятся в annotated/
```

Если рядом с изображениями лежит `bounding_boxes.json` (его создаёт
генератор синтетики), детектор автоматически считает precision / recall
/ F1 при `IoU ≥ 0.3`.

### Ожидаемые метрики (на синтетике 50 фото, seed=42)
- Изображений: 50
- Всего стаканов (GT): 87
- Precision: **~0.98**
- Recall: **~0.61**
- F1: **~0.75**

Это **baseline для fallback-режима**, не заменяет fine-tuned YOLOv8.
Главные ограничения:
- Плохо справляется, когда цвет фона почти идентичен цвету стакана
  (бежевый стакан на бежевом столе) — даже edge-based fallback находит
  не все такие стаканы.
- Не различает классы (cup / mug / can) — только «похоже на бумажный
  стакан по цвету и форме».
- Не используется для продакшена; цель — иметь хоть какой-то детектор,
  пока нет обученной модели.

### Когда переключаться обратно на YOLO?
Как только появится возможность поставить `ultralytics` (или обученную
модель `best.pt`) — используйте `baseline_test.py` и `infer.py`.
Лёгкий детектор оставьте как fallback для CI-тестов и отладки без GPU.
