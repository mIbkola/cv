# ТЗ: VOICE — Голосовой модуль (клон голоса С. Бурунова + RAG анекдотов 1986)

## 1. Цель
Робот должен уметь:
1. **Говорить** голосом, похожим на Сергея Бурунова.
2. **Рассказывать анекдоты 1986 года** (и вообще анекдоты соответствующей эпохи) по голосовой команде или триггеру.
3. Работать в реальном времени на борту робота (Jetson Orin) или оффборд-сервере с минимальной задержкой.

## 2. Компоненты

### 2.1. TTS (Text-to-Speech) — клон голоса Бурунова
**Подход:**
- Базовая модель: **XTTS v2** (Coqui) / **F5-TTS** / **Silero** — выбрать по качеству клонирования на русском
- Датасет: коллекция аудио/видео с голосом С. Бурунова (≥ 5–10 минут чистого голоса)
- Пайплайн:
  1. ASR Whisper — выровнять транскрипты
  2. VAD + noise gate — нарезать чистые фразы
  3. Fine-tune TTS на 5–15 эпохах
  4. Тестовое прослушивание + MOS-оценка

**Требования:**
- RTF (real-time factor) ≤ 1.0 на Jetson Orin
- Частота дискретизации 22 050 Гц (минимум 16 000)
- Поддержка русского языка без артефактов

### 2.2. RAG-система анекдотов
**Назначение:** по запросу «расскажи анекдот из 86-го» / «анекдот про штирлица» / и т.п. — извлекать релевантный анекдот из базы и подавать его в TTS.

**Компоненты:**
- **База анекдотов:** сборник анекдотов 1980-х (1986 ± 5 лет), парсинг из открытых источников / ручная подборка
- **Эмбеддинги:** `sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2` или `intfloat/multilingual-e5-small`
- **Векторная БД:** Qdrant (локально) или FAISS
- **Ранкер:** опционально cross-encoder
- **LLM-обертка (опционально):** для переформулирования / адаптации анекдота под контекст (например, GPT-4o-mini / локальная Qwen2.5-7B-Instruct)

**Пайплайн:**
```
[voice command / text] → ASR (если voice) → LLM intent ("joke", topic="штирлиц", year="1986")
   → RAG retrieve top-k → rerank → выбранный текст → TTS (Бурунов) → speaker
```

### 2.3. Интеграция с роботом
- Вход: микрофон на борту (RealSense D435 имеет встроенный микрофон) или внешний USB-микрофон
- Выход: динамик (USB / 3.5 jack, не входит в комплект G1 — нужен внешний)
- Управление паузами / прерываниями: если робот начинает движение — голос можно приглушить или поставить на паузу

## 4. Критерии приёмки
- [ ] TTS на ≥ 80% тестовых фраз неотличим от оригинала (слепое тестирование, 3+ слушателя)
- [ ] RAG возвращает релевантный анекдот в топ-3 в ≥ 90% случаев на тестовом наборе
- [ ] Полный цикл (голосовой запрос → ответ голосом) ≤ 5 сек на Jetson Orin
- [ ] Анекдоты 1986 года представлены в базе ≥ 50 шт.

## 5. Риски
- **Авторские права:** голос С. Бурунова — ограниченное использование, только для внутреннего проекта
- **Качество аудио датасета:** если исходники плохие — клон будет «картавить»
- **Русский язык в TTS:** XTTS работает, но требует проверки интонации
- **Анекдоты 1986:** возможно придется собирать вручную из печатных источников
