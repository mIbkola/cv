# ТЗ: COFFEE — Робот приносит кофе

## 1. Цель
Робот слышит/получает команду **«принеси кофе Олегу»** и должен:
1. Найти стакан с кофе (предположительно бумажный) в окружении.
2. Подойти к нему.
3. Взять стакан **не раздавив** и **не разлив**.
4. Доставить Олегу.

## 3. Подзадачи

### 3.1. COFFEE-detection — CV модуль
**Задача:** обнаружить бумажный стакан с кофе в кадре, выдать 3D-координаты.

**Подход:**
1. **Pre-trained baseline:** YOLOv8 / RT-DETR на COCO
   - Классы COCO: `cup` (id=41), но он не различает бумажный стакан и кружку
2. **Проверка качества:** проверить на тестовых кадрах с бумажными стаканами
3. **Если плохой результат:** fine-tune на своем датасете
   - Собрать 200–500 фото бумажных стаканов с кофе разных брендов
   - Разметка: bounding box + опционально сегментация
   - Аугментации: освещение, ракурс, наличие пара
4. **3D-позиционирование:** RealSense D435 → depth map → 3D-координаты стакана в системе координат робота

**Важно:** CV не должен быть единственным источником истины при хвате. На финальном этапе — переключение на force-feedback от пальцев.

### 3.2. COFFEE-manipulation — руки + сенсоры
**Задача:** взять стакан не раздавив (бумажный стакан легко мнется) и не разлив (открытый верх).

**Аппаратные возможности:**
- RH56DFTP: 12 суставов, 6 DoF, сенсоры усилия 10–2500 г
- Макс. нагрузка на руку 3 кг (стакан с кофе ≈ 200–400 г — запас есть)
- Длина руки 0.45 м

**Подход:**
1. **Грубое наведение:** по 3D-координатам от CV подвести руку к стакану
2. **Плавное сближение:** скорость ≤ 5 см/с на последних 10 см
3. **Force-guided grasp:**
   - Пальцы расходятся, медленно сходятся
   - Контроль силы прижима через сенсоры RH56DFTP
   - **Порог срабатывания:** 30–80 г нормальной силы (бумажный стакан начинает деформироваться при ~150 г)
4. **Stability check:** поднять стакан на 5 см, проверить смещение по силе → если изменилось > 20% — перехватить
5. **Фазы переноса:**
   - Удержание вертикально (IMU руки + замкнутый контур силы)
   - Компенсация покачивания тела при ходьбе

**Безопасность:**
- Если сила > 200 г на пальце → мгновенный разжим (anti-crush)
- Если стакан проскальзывает (дельта силы > 30% за 200 мс) → перехват или возврат на базу

### 3.3. COFFEE-locomotion — ходьба
**Задача:** дойти до стакана, потом дойти до Олега, неся стакан.

**Используемые средства:**
- `unitree_sdk2py` → `SportClient` высокоуровневое управление
- 3D Lidar Livox MID-360 + RealSense D435 → построение карты / obstacle avoidance
- Скорость снижена до 0.5–0.8 м/с (вместо 2 м/с) для устойчивости с грузом

**Фазы:**
1. **Scan:** построить карту окружения, найти стакан (CV) — `(x, y, z)`
2. **Approach:** проложить путь (A* / RRT / встроенный навигатор) → остановиться в 0.5 м от стакана
3. **Pick:** статичная поза, манипуляция
4. **Carry:** путь к Олегу (человек детекция — YOLO `person` или face detection)
5. **Handover:** остановиться в 0.7 м, поднять руку с стаканом на уровень груди Олега, дождаться приема (сила на пальцах упала < 20 г → отпустить)

### 3.4. COFFEE-llm_orchestrator — оркестратор
**Задача:** разобрать команду «принеси кофе Олегу» на subtasks и вызывать остальные модули.

**Варианты:**
- **Скрипт-стейт-машина** (Python): жёсткая логика, дёшево, предсказуемо
- **LLM-агент** (function calling): GPT-4o / Qwen2.5-7B + tool use — гибче, но дороже

**Гибрид:** LLM разбирает интент и параметры (кто = Олег, что = кофе), дальше — детерминированная стейт-машина.

```
state machine:
  IDLE → PARSE_CMD → FIND_CUP → APPROACH_CUP → GRASP → STABILITY_CHECK
       → FIND_TARGET (Олег) → APPROACH_TARGET → HANDOVER → RELEASE → IDLE
```

**Tools для LLM:**
- `find_object(class_name)` → CV
- `navigate_to(x, y)` → locomotion
- `grasp_with_force_feedback(target_pose)` → manipulation
- `verify_grasp()` → manipulation
- `find_person(name)` → CV
- `handover()` → manipulation + locomotion

## 5. Критерии приёмки
- [ ] CV находит бумажный стакан в кадре с точностью mAP@0.5 ≥ 0.85
- [ ] Робот подходит к стакану с точностью позиционирования ≤ 5 см
- [ ] Хватка стакана без раздавливания (деформация стакана < 5%)
- [ ] Перенос на 5 м без расплескивания (потери жидкости < 5 мл)
- [ ] Полный цикл «команда → передача Олегу» ≤ 90 сек
- [ ] Срабатывание anti-crush при ошибке ≥ 99%

## 6. Риски и mitigation
| Риск | Mitigation |
|---|---|
| Бумажный стакан легко мнется | Force feedback, мягкий хват, низкий порог срабатывания (30–80 г) |
| Кофе расплескивается при ходьбе | Плавная ходьба, держать стакан вертикально через IMU, снизить скорость |
| CV путает стакан с кружкой | Fine-tune на датасете бумажных стаканов + проверка формы (цилиндр, сужение книзу) |
| Олега нет в комнате | Timeout 30 сек → вернуть стакан на стол → сообщить голосом (через VOICE-модуль) |
| Сенсоры RH56DFTP неоткалиброваны (руки не родные) | Обязательная калибровка перед запуском, тарировка сенсоров на эталонных весах |
| Робот падает с горячим кофе | Встроенный recovery (G1 умеет вставать) + проверка температуры через IR-датчик (если есть) |

## 7. Интеграция с VOICE-модулем
- Команда «принеси кофе Олегу» может прийти голосом → ASR → orchestrator
- Статусные сообщения: «Иду за кофе», «Не нашёл стакан», «Олег, держи» → через TTS (Бурунов)
